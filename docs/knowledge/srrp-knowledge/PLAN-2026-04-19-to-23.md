## 📅 Faz 1 — Pazar (20 Nisan): Backend Veri Altyapısı

**Hedef:** Tek güvenilir veri kaynağı. Frontend'in geri kalanı buna yaslanabilsin.

### 1.1 Saatlik Scheduler (APScheduler)
- `backend/services/scheduler.py` — `BackgroundScheduler`, `CronTrigger(minute=0)`.
- İş: 81 il × 3 metrik (rüzgar/sıcaklık/ışınım) Open-Meteo hourly fetch → `HourlyWeatherData` tablosuna upsert.
- `last_scheduler_run` timestamp'i meta tabloda.
- Başlatma: FastAPI `startup` event.
- Logging: her çalışma `ok/fail` + süre.

### 1.2 `province_analysis` Tablosu
- Alanlar: `province_name`, `resource_type` (wind/solar/hydro), `score_1m`, `score_3m`, `score_6m`, `score_yearly`, `computed_at`.
- **4 zaman penceresi:** 30 / 90 / 180 / 365 gün. `score_3m` (90 gün) Önerilen Bölgeler "1-6 ay kısa vadeli" spec'i için kritik ara pencere — 2026-04-19'da eklendi.
- Ham metrikler: `avg_wind_speed`, `avg_solar_radiation`, `avg_temperature`, `capacity_factor`, `sample_count` (debug/detay için).
- 10+ yıllık geçmiş + son çekim üzerinden hesaplama (mean, capacity factor, seasonal).
- Seed: migration + bir kereye mahsus full-history import script.
- Her saatlik scheduler tetiklemesinden sonra `score_*` yeniden hesaplanır (incremental).

#### Skor Fonksiyonları (2026-04-19 netleştirme)
- **Wind** (`wind_speed_100m` ortalaması → 0-100): **Cube-law default** (P ∝ v³), lineer opsiyonel. Cut-in 3 m/s → 0, rated 10 m/s → 100.
- **Solar** (`shortwave_radiation` ortalaması → 0-100): Üst sınır **400 W/m²** (500 yüksekti — TR ortalaması 150-300). 0 → 0, 400 → 100.
- **Hydro** (`precipitation` toplamı + `temperature_2m` faktörü → 0-100): Basit proxy. TODO: HES havza verisi eklenince revize.

### 1.3 Endpoint'ler (tek kaynak)
- `GET /analysis/provinces?type=wind&horizon=6m` → top-N + tüm liste. `horizon` kabul eder: `1m` | `3m` | `6m` | `yearly`.
- `GET /analysis/province/{name}` → İl Analizi için detay.
- `GET /reports/provinces` → Raporlar için.
- `GET /choropleth/{metric}` → Harita renk dağılımı (`province_analysis`'ten).
- **Eski "canlı hesaplama" endpointlerini deprecate et** (eski kodu silmek zorunda değilsin, ama UI yeniye geçsin).

### 1.4 Timestamp / "228 dk önce" Fix
- `GET /system/status` → `last_scheduler_run`, `next_scheduler_run`.
- Frontend `MapScreen` üstündeki "X dk önce güncellendi" gerçek değer.

**Çıktı:** Saatlik güncellenen, tek kaynaklı, doğrulanmış veri. Frontend henüz değişmedi.
